Getting started with Microsoft Azure

1 Introduction

We will use the Azure Portal (a browser-based user interface) to create a Linux virtual machine running Ubuntu 20.04 LTS. We will then connect to the virtual machine via SSH and install an NGINX web server. Finally, we will use the browser to connect to this server.

2 Creating the virtual machine

  1. Log in to the Azure portal using your CentraleSupélec credentials.

  2. Select Virtual Machines.

  3. Click on Create and then Azure Virtual Machine.

  4. Create a new resource group and give it a name of your choice.

  5. Assign a name to your virtual machine.

  6. Select (Europe) France Central as the region. If that region does not work for you, try another one.

  7. Accept the default settings for availability zone and security type.

  8. Select Ubuntu Server 20.04 LTS - Generation 2 as the image.

  9. Select Standard_B1s - 1 virtual CPU, 1 GiB memory as the Size, if available. Otherwise, choose the closest or most affordable configuration.

  10. Choose SSH public key as the authentication type.

  11. Choose a username.

  12. Accept the default SSH key options.

  13. Under inbound ports, select SSH (22) and HTTP (80).

  14. Click on the button Review + Create and then Create. You will be asked to download the private key. Accept and save the key in a folder of your choice.

3 Connect to the virtual machine

  1. Click on Go to Resource to view your virtual machine’s details.

  2. Copy the public IP address.

  3. Enter the following command to change the permissions of your private key. The private key is stored in a .pem file that you downloaded earlier.

chmod 644 <path_to_pem_file>
  1. Enter the following command to connect to your virtual machine.
ssh -i <path_to_pem> <username>@<vm_ip_address>

4 Install the Nginx Web Server

Simply enter the following two commands in the terminal of the virtual machine:
sudo apt-get -y update
sudo apt-get -y install nginx

Once the installation is complete, open the web browser of your choice and enter the virtual machine’s IP address. You should see the default NGINX welcome page.

5 Cleaning up resources

Don’t forget to stop the virtual machine or your credit will be used up! Since we will no longer be using this virtual machine, it is recommended to delete the entire resource group created during this activity.
