Gestion et analyse des données — Tutoriel 2

SQL

L’objectif de ce tutoriel est de pratiquer les requêtes SQL sur une base de données relationnelle.

Préparation de l’environnement

  • Démnarrez la machine virtuelle fournie pour le cours avec Remote Desktop Connection.

  • Ouvrez pgAdmin4 (icône sur le bureau).

  • Dans le menu de gauche, faites un clic droit sur Servers et sélectionnez PostgreSQL 18 pour vous connecter.

  • Lorsque vous y êtes invité, entrez le mot de passe postgres pour vous connecter au serveur.

  • Sélectionnez la base de données travel.

  • Ouvrez un nouvel outil de requête (clic droit sur la base de données travel, sélectionnez Query tool).

Schéma de la base de données

La figure 1 montre le schéma physique de la base de données travel. Vous pouvez également l’obtenir dans pgAdmin4 en faisant un clic droit sur la base de données travel, et en sélectionnant ERD for database.

Figure 1: Le schéma physique de la base de données travel

Contraintes de clé étrangère

Regardez le schéma physique de la base de données travel. Les tables sont liées par des contraintes de clé étrangère.

Par exemple, la table flight_booking a une colonne trip_id qui référence la colonne trip_id dans la table trip.

Exercice

Exercise 1 Essayez de supprimer une ligne de la table trip qui est référencée par une ligne dans la table flight_booking. Que se passe-t-il?

Solution

D’abord, nous devons identifier une ligne dans la table trip qui est référencée par une ligne dans la table flight_booking.

SELECT * 
FROM flight_booking JOIN trip USING (trip_id)
LIMIT 1;

Le trip_id 0 est obtenu à partir du résultat de la requête précédente.

Si on essaie de supprimer le trip 0 de la table trip :

DELETE FROM trip
WHERE trip_id = 0;

Nous obtenons l’erreur suivante :

ERROR:  update or delete on table trip violates foreign key constraint flight_booking_trip_id_fkey on table flight_booking
Key (trip_id)=(0) is still referenced from table flight_booking.

Par défaut, les contraintes de clé étrangère empêchent la suppression d’une ligne qui est référencée par une autre ligne dans une table différente.

Regardons comment la contrainte de clé étrangère est définie dans la table flight_booking.

  • Dans pgAdmin4, dans le menu de gauche, développez la base de données travel > Schéma public > Tables.
  • Faites un clic droit sur la table flight_booking.
  • Sélectionnez Properties.
  • Sélectionnez l’onglet Constraints.
  • Sélectionnez l’onglet Foreign Key.
  • Sélectionnez la contrainte de clé étrangère nommée flight_booking_trip_id_fkey.
  • Cliquez sur l’icône en forme de crayon pour modifier la contrainte.
  • Sélectionnez l’onglet Actions.

Ici vous pouvez voir que l’action de suppression est définie sur No Action.

Exercice

Exercise 2 Comment l’action de mise à jour est-elle définie pour la contrainte de clé étrangère flight_booking_trip_id_fkey dans la table flight_booking?

Essayez maintenant de modifier le trip_id d’une ligne de la table trip qui est référencée par une ligne de la table flight_booking.

Que se passe-t-il?

Solution

L’action de mise à jour est définie par Cascade. Cela signifie que si nous mettons à jour la valeur de la clé primaire dans la table trip, les valeurs correspondantes de la clé étrangère dans la table flight_booking seront automatiquement mises à jour.

Remplaçons le trip_id 0 par 5432 dans la table trip:

UPDATE trip
SET trip_id = 5432
WHERE trip_id = 0;

Maintenant la valeur correspondante de la clé étrangère dans la table flight_booking a été automatiquement mise à jour.

Requêtes SQL exploratoires

Lorsque vous travaillez avec une nouvelle base de données, il est important d’explorer les données pour se familiariser avec sa structure et son contenu. Nous pouvons utiliser des requêtes SQL pour effectuer cette exploration.

Exercice

Exercise 3

  1. Combien de clients sont des hommes et combien sont des femmes ?

  2. Quel est le nombre de clients dans chaque pays ?

  3. Quel est l’âge moyen des clients dans chaque pays ?

  4. Quel est le nombre total de réservations d’hôtel effectuées pour chaque hôtel ? Triez les résultats par ordre décroissant.

  5. Quel est le montant total dépensé pour les réservations d’hôtel par chaque client ? Triez les résultats par ordre décroissant et affichez uniquement les 10 meilleurs clients.

  6. Combien d’aéroports y a-t-il dans chaque pays ? Listez uniquement les pays ayant plus de 10 aéroports.

Solution
  1. Combien de clients sont des hommes et combien sont des femmes ?
SELECT gender, COUNT(*) AS num_customers
FROM customer
GROUP BY gender
  1. Quel est le nombre de clients dans chaque pays ?
SELECT country, COUNT(*) AS num_customers
FROM customer
GROUP BY country
  1. Quel est l’âge moyen des clients dans chaque pays ?
SELECT country, AVG(EXTRACT(YEAR FROM AGE(CURRENT_DATE, birth_date))) AS avg_age
FROM customer
GROUP BY country
  1. Quel est le nombre total de réservations d’hôtel effectuées pour chaque hôtel ? Triez les résultats par ordre décroissant.
SELECT hotel_id, COUNT(*) AS num_reservations
FROM hotel_booking
GROUP BY hotel_id
ORDER BY num_reservations DESC
  1. Quel est le montant total dépensé pour les réservations d’hôtel par chaque client ? Triez les résultats par ordre décroissant et affichez uniquement les 10 meilleurs clients. =
SELECT customer_id, SUM(price) AS total_spent
FROM hotel_booking JOIN trip USING (trip_id)
GROUP BY customer_id
ORDER BY total_spent DESC
LIMIT 10
  1. Combien d’aéroports y a-t-il dans chaque pays ? Listez uniquement les pays ayant plus de 10 aéroports.
SELECT country, COUNT(*) AS num_airports
FROM airport
GROUP BY country
HAVING COUNT(*) > 10

Requêtes métier

Maintenant, imaginons que vous travaillez pour l’agence de voyage qui gère la base de données. On vous a demandé de fournir quelques informations basées sur les données stockées dans la base de données.

Exercice

Exercise 4

  1. Identifiez les 5 destinations les plus populaires en fonction du nombre de vols réservés.

  2. Trouvez la durée moyenne des séjours pour chaque hôtel.

  3. Déterminez les 10 clients qui ont effectué le plus grand nombre de réservations (hôtel et vol).

  4. Analysez les tendances mensuelles des réservations de vols et des réservations d’hôtels.

  5. Affichez les détails du voyage numéro 1. En particulier, listez les prénoms et noms du client, et, pour chaque vol, les aéroports de départ et d’arrivée, la date de départ, et les villes de départ et d’arrivée des aéroports.

Solution
  1. Identifiez les 5 destinations les plus populaires en fonction du nombre de vols réservés.
SELECT airport_dst, city, country,  COUNT(*) AS num_bookings
FROM flight_booking JOIN flight USING (flight_id)
        JOIN airport ON airport_id=airport_dst
GROUP BY airport_dst, city, country
ORDER BY num_bookings DESC
LIMIT 5
  1. Trouvez la durée moyenne des séjours pour chaque hôtel.
SELECT hotel_id, AVG(CHECKOUT_DATE - CHECKIN_DATE) AS avg_stay_duration
FROM hotel_booking
GROUP BY hotel_id
  1. Déterminez les 10 clients qui ont effectué le plus grand nombre de réservations (hôtel et vol).
SELECT customer_id, first_name, last_name, COUNT(*) AS num_bookings
FROM (
    SELECT customer_id, first_name, last_name
    FROM flight_booking JOIN trip USING (trip_id) JOIN customer USING(customer_id)
    UNION ALL
    SELECT customer_id, first_name, last_name
    FROM hotel_booking JOIN trip USING (trip_id) JOIN customer USING(customer_id)
) AS all_bookings
GROUP BY customer_id , first_name, last_name
ORDER BY num_bookings DESC
LIMIT 10
  1. Analysez les tendances mensuelles des réservations de vols et des réservations d’hôtels.
-- Flight bookings by month
SELECT EXTRACT(MONTH FROM departure_date) AS month, COUNT(*) AS num_flights
FROM flight_booking JOIN flight USING (flight_id)
GROUP BY month
ORDER BY month;

-- Hotel reservations by month
SELECT EXTRACT(MONTH FROM check_in_date) AS month, COUNT(*) AS num_reservations
FROM hotel_booking
GROUP BY month
ORDER BY month;
  1. Affichez les détails du voyage numéro 1. En particulier, listez les prénoms et noms du client, et, pour chaque vol, les aéroports de départ et d’arrivée, la date de départ, et les villes de départ et d’arrivée des aéroports.
SELECT c.first_name,
        c.last_name, 
        s.airport_name as departure_airport, 
        concat(s.city, ', ', s.country) as departure_loc, 
        departure_date,
        d.airport_name as arrival_airport,
        concat(d.city, ', ', d.country) as arrival_loc
FROM customer c
    JOIN trip t USING (customer_id)
    JOIN flight_booking USING (trip_id)
    JOIN flight f USING(flight_id)
    JOIN airport s ON s.airport_id = f.airport_src 
    JOIN airport d ON d.airport_id = f.airport_dst
WHERE t.trip_id=1
