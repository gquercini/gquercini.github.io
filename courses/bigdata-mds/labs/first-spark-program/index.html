Big Data MDS — Lab 2

Premier programme Spark sur DCE

Le Data Centre d’Enseignement (DCE) est un ensemble de ressources informatiques financées par l’Eurométropole de Metz par l’Eurométropole de Metz, la Région Grand Est, CentraleSupélec et sa fondation, et le Conseil Départemental de Moselle.

Ce tutoriel est un guide rapide pour :

  • Apprendre les commandes de base pour manipuler les fichiers stockés dans HDFS.

  • Apprendre à exécuter un programme Spark sur le DCE.

Plus d’informations sont disponibles sur le site officiel du DCE.

1 Présentation du DCE

L’architecture du DCE est présentée en Figure 1.1.

Figure 1.1: Architecture du cluster (source : documentation du DCE)

Dans ce cours, nous allons utiliser uniquement les noeuds CPU. Ces derniers sont divisés en deux groupes: le cluster Sarah et le cluster Kyle.

2 Exécution d’un programme Spark

Les datasets sont normalement disponibles dans le dossier /data/ stocké dans HDFS.

Saisissez la commande suivante pour afifcher le contenu du dossier :

hdfs dfs -ls -h hdfs://sar01:9000/data

L’objectif ici est d’exécuter un code Spark pour compter le nombre d’occurrences de chaque mot dans le fichier /data/sherlock.txt.

  1. Copiez le fichier ~cpu_vialle/DCE-Spark/template_wc.py vers votre dossier personnel en saisissant la commande suivante :

cp ~cpu_vialle/DCE-Spark/template_wc.py ./wc.py

Saisissez la commande ls pour vérifier que le fichier wc.py figure bien dans votre dossier personnel. Ce fichier contient le code Python du programme. Ce fichier devrait aussi apparaître dans l’explorateur de Visual Studio Code ; si cela n’est pas le cas, cliquez sur le bouton d’actualisation de l’explorateur.

  1. Ouvrez le fichier wc.py dans Visual Studio Code.

  2. Recherchez l’instruction suivante :

text_file = sc.textFile("hdfs://...")

et remplacez-la avec la suivante :

text_file = sc.textFile("hdfs://sar01:9000/data/sherlock.txt")

Cette instruction créera un RDD nommé text_file avec le contenu du fichier.

  1. De la même manière, cherchez l’instruction suivante :

counts.saveAsTextFile("hdfs://...")

et remplacez-la par la suivante :

counts.saveAsTextFile("hdfs://sar01:9000/itpspark24/itpspark24_X/sherlock.out")

Cette instruction créera un dossier sherlock.out qui contiendra les fichiers renvoyés par le programme.

  1. Exécutez le programme wc.py à l’aide de la commande suivante :

spark-submit --master spark://sar01:7077 wc.py

  • Lorsque l’exécution est terminée, la sortie sera disponible dans le dossier sherlock.out. Pour le vérifier, exécutez la commande suivante :

hdfs dfs -ls -h hdfs://sar01:9000/itpspark24/itpspark24_X/sherlock.out

Comme d’habitude, n’oubliez pas de remplacer itpspark24_X par votre nom d’utilisateur.

  1. Pour afficher le résultat, exécutez la commande suivante :

hdfs dfs -cat hdfs://sar01:9000/itpspark24/itpspark24_X/sherlock.out/*

Fichiers de sortie

Si vous réexécutez le programme en spécifiant un fichier de sortie qui existe déjà, vous obtiendrez une erreur. Si vous voulez vraiment écraser le fichier de sortie, vous devez d’abord le supprimer explicitement en tapant la commande suivante :

hdfs dfs -rm -r hdfs://sar01:9000/itpspark24/itpspark24_X/sherlock.out
