1 Calcul des moyennes

Nous considérons une collection de fichiers CSV contenant des mesures de température au format suivant :

année, mois, jour, heure, minute, seconde, température

vous pouvez trouver les fichiers dans le répertoire hdfs://sar01:9000/data/temperatures/

Voici les détails de chaque fichier :

  • Le fichier temperatures_86400.csv contient une mesure par jour pour les années 1980 - 2018.
  • Le fichier temperatures_2880.csv contient une mesure toutes les 2880 secondes pour les années 1980 - 2018.
  • Le fichier temperatures_86.csv contient une mesure toutes les 86 secondes pour la seule année 1980.
  • Le fichier temperatures_10.csv contient une mesure toutes les 10 secondes pour les années 1980 - 2018.

1.1 Première implémentation

Copiez le fichier ~vialle/DCE-Spark/template_temperatures.py dans votre répertoire personnel en saisissant la commande suivante :

cp ~vialle/DCE-Spark/template_temperatures.py ./avg_temperatures_slow.py

Ouvrez le fichier avg_temperatures_slow.py. Le fichier contient l’implémentation de la fonction avg_temperature_slow qui:

  • prend en arguments un RDD, dont chaque élément est une ligne du fichier donné ;

  • renvoie un RDD, où chaque élément est une paire clé-valeur (year, avg_temp).

Dans le même fichier, localisez les deux variables input_path et output-path et écrivez le code suivant :

input_path = "hdfs://sar01:9000/data/temperatures/"
output_path = "hdfs://sar01:9000/archimat25/archimat25_X/"

Remplacez X par le numéro correspondant à votre compte ! N’oubliez pas le / à la fin des chemins d’accès aux fichiers !

Faites les actions suivantes :

  • Exécutez le script avg_temperatures_slow.py en utilisantle fichier temperatures_86400.csv comme argument. Pour ce faire, utilisez la commande suivante :

spark-submit --master spark://sar01:7077 avg_temperatures_slow.py temperatures_86400.csv

  • Vous devriez trouver la sortie du programme dans le dossier suivant :

hdfs://sar01:9000/archimat25/archimat25_X/temperatures_86400.out

Tapez la commande suivante pour le vérifier :

hdfs dfs -ls hdfs://sar01:9000/archimat25/archimat25_X/temperatures_86400.out

  • Si vous souhaitez accéder au résultat du calcul, vous pouvez exécuter la commande suivante :

hdfs dfs -cat hdfs://sar01:9000/archimat25/archimat25_X/temperatures_86400.out/*

1.2 Deuxième implémentation

Nous voulons mettre en œuvre une meilleure version du programme. Vous pouvez rédiger vos idées sur papier avant d’écrire du code.

Lorsque vous êtes prêts, créez une copie du fichier avg_temperatures_slow.py et renommez-la en avg_temperatures_fast.py, avec la commande suivante :

2 Amis communs dans un réseau social

Considérons un réseau social décrit par un graphe encodé dans un fichier texte. Chaque ligne du fichier est une liste d’identifiants séparés par des virgules. Par exemple, la ligne \(A,D,C,B\) signifie que \(A\) est ami avec \(D\), \(C\) et \(B\). Un extrait du fichier ressemble à ce qui suit :

B,D,A
A,D,C,B
D,A,C,B
C,A,D
...

Nous supposons que la relation d’amitié est symétrique : \((A, B)\) implique (B, A)$.

Nous voulons obtenir la liste des amis communs pour chaque paire d’individus :

(B, C), [A, D] 
(A, D), [B, C] 
(C, D), [A]
(A, C), [D] 
(B, D), [A] 
(A, B), [D]

Comme contrainte supplémentaire, nous voulons représenter un couple une seule fois et éviter de représenter un couple symétrique. En d’autres termes, si nous produisons \((A, B)\), nous ne voulons pas produire \((B, A)\).

Nous utilisons les fichiers suivants, disponibles dans le dossier hdfs://sar01:9000/data/social-network/:

  • sn_tiny.csv. Petit réseau social, que vous pouvez utiliser pour tester votre implémentation.

  • sn_10k_100k.csv. Réseau social avec \(10^4\) individus et \(10^5\) liens.

  • sn_100k_100k.csv. Réseau social avec \(10^5\) individus et \(10^5\) liens.

  • sn_1k_100k.csv. Réseau social avec \(10^3\) individus et \(10^5\) liens.

  • sn_1m_1m.csv. Réseau social avec \(10^6\) individus et \(10^6\) liens.

2.1 Implémentation

Obtenez un squelette du code à l’aide de la commande suivante :
