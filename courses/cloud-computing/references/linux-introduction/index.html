1 Introduction

Linux is a family of open source operating systems that has its roots in Unix, an operating system developed in the 1970s at the Bell Labs research center by, among others, Ken Thompson and Dennis Ritchie. For this reason, Linux is said to be a Unix-like operating system.

A Unix-like operating system is composed of three parts:

  1. The kernel. It’s the core of the operating system, acting as an intermediary between the applications and the hardware devices. Its tasks include memory management, process scheduling and input-output management.

  2. The shell. This is the command-line terminal that acts as an interface between the user and the kernel. The shell interprets and executes the commands typed by the user. Many variants exist of the shell; the default shell provided by most Linux distributions is called Bash. Usually, a shell provides some useful functionalities to the user, including filename completion (one can type part of the name of a command or a file and press the Tab key to have the shell complete it) and the history (list of all commands typed by the user, so if the user needs to retype a command, s/he can just use the cursor keys to retrieve it in the history).

  3. The applications. These are the user programs running on top of the operating system (e.g., Firefox).

The first Linux kernel has been released as an open source software by Linus Torvalds on September 17, 1991. Being open source, the source code can be read, modified and distributed by anyone under the GNU General Public License. This has been key to the success and the rapid adoption of Linux. Since then, many different Linux distributions have been developed and made available for free; all distributions are based on the Linux kernel, they mainly differ in the applications and tools they provide. Some of the most popular are Ubuntu, Debian and Fedora.

2 The directory structure

In Linux all files are organized in a hierarchical file system that can be represented as a tree (Figure 2.1). At the top of the hierarchy stands a special directory called the root, that is usually written as a slash (/).

Figure 2.1: Linux directory structure

The root directory includes many special directories, such as boot (contains configuration files necessary to boot Linux), lib (contains shared library files necessary to boot the system), bin (contains user executable files), dev (contains files that represent hardware devices) and etc (contains the local system configuration files).

Each user is assigned a special directory called home (i.e., laura, michel) that is stored under the home directory; despite its name, the directory named root is not the root directory, but the home directory of the user root, that is the administrator of the system. Each directory (except the root directory /) has one parent directory and zero or many children directories. For instance, the parent directory of home is the root directory / and its children directories are laura and michel.

When a user (e.g., Laura) logs in the system and opens a shell, her working directory (i.e., the directory where she finds herself in) is her home directory. From there, Laura can type any Linux command and reference any file or directory in the file system (if she has the necessary rights). In Linux, Laura has two ways to specify the name of a directory or a file: absolute paths or relative paths.

The absolute path to a file or directory is the full path of that file or directory in the tree hierarchy starting from the root. If Laura types the command pwd (i.e., print working directory), as in Figure 2.2, she’ll get the absolute path /home/laura of her home directory.

Figure 2.2: Home and current directory

Referring to Figure 2.1:

  • the absolute path of the file meeting.txt is /home/laura/work/meeting.txt;
  • the absolute path of the file family.jpg is /home/laura/personal/family.jpg;
  • the absolute path of the directory michel is /home/michel.

The relative path to a file or a directory is given with respect to the working directory. In order to specify a relative path, Laura might need to use the symbol “.”, that indicates the working directory, and “..”, that indicates the parent of the working directory. If Laura wants to reference the file family.jpg and her working directory is /home/laura, she’ll type ../personal/family.jpg. If Laura wants to reference the file meeting.txt, she’ll type ./meeting.txt, or simply meeting.txt, as any path is relative to the current directory, if Laura doesn’t specify otherwise.

Exercise

Exercise 2.1 If Laura’s current directory is /etc, what would
the relative path of the file /home/laura/work/meeting.txt be?

Solution

../home/laura/work/meeting.txt

3 Useful Linux commands

Linux provides many useful commands to navigate the file system and manipulate files and directories.

3.1 pwd

pwd stands for print work directory. It returns the absolute path of the working directory.

3.2 ls

Used to list the content of the working directory. The command has several options, in particular:

  • ls -l. Show a detailed view of the content of the working directory.
  • ls -a. Includes the hidden files in the working directory. In Linux, all files whose name begins with “.” are hidden.

3.3 cd

cd stands for change directory; the command is used to change working directory.

If Laura’s working directory is /home/laura and she wants to move to the directory work she’ll type the following command:

cd work

If she wants to return to her home directory, she’ll type:

cd ..

3.4 mkdir

mkdir stands for “make directory”. This command is used to create a new directory.

Suppose that Laura’s working directory is /home/laura; in order to create a new directory /home/laura/videos, she’ll type:

mkdir videos

If she wants to create a new directory /home/laura/videos/birthday, she’ll type:

mkdir videos/birthday

3.5 rm

This command is used to remove a file or a directory. Suppose Laura’s working directory is /home/laura. In order to remove the file meeting.txt, she’ll type:

rm meeting.txt

If she wants to remove a directory (for example videos), she’ll type:

rm -r videos

This will delete the directory videos and all its content.

3.6 cat

This command is useful for different tasks. One possible use is to show the content of a file. If Laura types:

cat meeting.txt

she’ll get the content of the file meeting.txt.

3.7 echo

This command writes to the screen whatever Laura gives it as a parameter. If Laura types:

echo hello world

the command will print the string “hello world”. Although this might not sound very useful, the command echo comes in handy when combined with others. For instance, if Laura types:

echo hello world > hello.txt

a new file named hello.txt is created in the working directory containing the string “hello world”. This is a very useful way to create a new file with some content.

3.8 cp

This command is used to create a copy of a file or directory. Suppose that Laura’s working directory is /home/laura. If she types:

cp meeting.txt ./personal

a new copy of meeting.txt is created in the directory /home/laura/personal.

3.9 mv

This command is used to move a file or directory. Suppose that Laura’s current directory is /home/laura that contains two children directories named personal and videos. If Laura wants to move the directory videos under personal, she’ll type:

mv videos personal
